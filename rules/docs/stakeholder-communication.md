# Stakeholder communication

When writing for a stakeholder audience:
- Lead with the decision or recommendation, not the methodology.
- State uncertainty explicitly: "The model is estimated to improve conversion by 3–7% (90% CI)" not "the model improves conversion."
- Do not include raw model output, confusion matrices, or p-values without plain-language interpretation alongside them.
- If the model cannot answer the stated question reliably, say so. Do not present a technically valid result as business-actionable when it is not.
- Avoid jargon without definition. If a technical term must appear, define it on first use.
